# Code Review и исправление кода

Ниже дан небольшой HTTP-сервис на Go. Его писал начинающий разработчик.

Что нужно сделать:

1. Внимательно прочитать код `main.go`.
2. Найти потенциальные проблемы:
   - гонки данных (data race),
   - проблемы с горутинами,
   - странные места в работе с context,
   - обработку ошибок,
   - другие подозрительные моменты.
3. Объяснить голосом, что именно не так и почему это может привести к проблемам в продакшене.
4. По возможности, в лайве поправить код, чтобы:
   - не было data race,
   - не было утечек горутин,
   - корректно использовались контексты,
   - код стал чуть аккуратнее.

Можно запускать код, добавлять свои функции, структуры, тесты - всё, что посчитаешь нужным.
